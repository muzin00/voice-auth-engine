# ステージング済みの差分からコミットメッセージを生成してコミットする

## 手順

1. `git diff --cached` でステージング済みの差分を取得する
2. 差分がない場合はその旨を伝えて終了する
3. 変更内容を分析し、コミットメッセージを生成する
4. 生成したコミットメッセージをユーザーに提示し、確認を取る
5. 承認後、`git commit` を実行する

## コミットメッセージの形式

- 1行目: 変更の要約（Conventional Commits 形式: `feat:`, `fix:`, `refactor:`, `chore:`, `docs:` など）
- 必要に応じて空行の後に補足説明を追加
