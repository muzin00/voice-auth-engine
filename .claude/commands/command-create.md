# Claude Code コマンドを対話形式で作成する

## ヒアリング

ユーザーに以下を質問（AskUserQuestion ツールを使用）:

- **コマンド名**: スラッシュコマンドの名前（例: test, lint, deploy）。先頭の `/` は不要。サブディレクトリ可（例: `db/migrate`）
- **目的**: このコマンドが達成する1つの目的
- **動作内容**: コマンド実行時にClaudeに何をさせるか

## コマンドファイルの生成

ヒアリング内容をもとに `.claude/commands/<コマンド名>.md` を作成する。
サブディレクトリが含まれる場合は、必要なディレクトリも作成すること。

### プロンプト記述ルール

- **先頭見出し (h1)**: CLI上にコマンドの説明として表示される。簡潔かつ明瞭にコマンドの目的を記述する
- **単一責任**: 1コマンド = 1つの目的のみ。複数のアクション（例: 実行→修正）は別コマンドに分離
- **冗長さの排除**: 背景説明・前置き・ペルソナ設定は書かない
- **文体**: 命令文・指示型で書く
- **構成**: 必要な箇所のみ見出し・リストを適度に使う
- **無駄の排除**: 行数制限は設けないが、不要な記述は削る

### `$ARGUMENTS` について

- ユーザーが渡す引数は `$ARGUMENTS` で参照できる
- 例: `/deploy staging` → `$ARGUMENTS` が `staging` に置換
- 引数が必要な場合のみプロンプト内に `$ARGUMENTS` を含める

## 確認

作成したコマンドの内容をユーザーに提示し、問題ないか確認する。
修正の要望があれば対応すること。
