# 現在のブランチから GitHub Pull Request を作成する

## 手順

1. `git log main..HEAD --oneline` と `git diff main...HEAD` でブランチの全変更を把握する
2. 変更内容を分析し、PRタイトルと本文を生成する
3. `gh pr create` でPRを作成する

## PRの形式

- **タイトル**: 70文字以内。変更の要約を簡潔に記述
- **本文**:

```
## 概要
<PRの背景・目的・変更内容を説明>
```

## 注意事項

- ベースブランチは `main` とする
- リモートに未プッシュのコミットがあれば先に `git push -u` する
- PRタイトル・本文は作成前にユーザーに確認する
